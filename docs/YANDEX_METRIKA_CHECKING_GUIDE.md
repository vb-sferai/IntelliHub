## Быстрая проверка (режим реального времени)

### Шаг 1: Откройте режим реального времени

1. Перейдите в **Яндекс.Метрику**: https://metrika.yandex.ru
2. Выберите счётчик **105383064** (SFER.AI)
3. В левом меню: **Отчёты** → **В реальном времени** → **Посетители онлайн**

### Шаг 2: Откройте сайт с UTM-метками

Откройте в новой вкладке:
```
https://ru.sfer.ai/?utm_source=test&utm_medium=manual&utm_campaign=metrika_check
```

### Шаг 3: Проверьте данные в Метрике

Через 10-30 секунд в отчёте появится ваш визит:

**Что вы увидите:**
- **Источник**: test
- **Канал**: manual
- **Кампания**: metrika_check
- **Страница**: ru.sfer.ai/

**✅ Если данные появились — UTM работают правильно!**

## Просмотр UTM-меток в отчётах

### 1. Отчёт "Метки UTM"

**Путь:** `Отчёты` → `Стандартные отчёты` → `Источники` → `Метки UTM`

Здесь вы увидите все переходы с UTM-метками:

#### Основные метрики:

| Столбец | Описание |
|---------|----------|
| **utm_source** | Источник трафика (yandex, google, telegram, vk) |
| **utm_medium** | Тип трафика (cpc, social, email, referral) |
| **utm_campaign** | Название кампании |
| **utm_term** | Ключевое слово (для контекстной рекламы) |
| **utm_content** | Вариант объявления |
| **Визиты** | Количество визитов с этими метками |
| **Конверсия** | % визитов, достигших цели |

#### Как пользоваться отчётом:

1. **Детализация**: кликните на любую строку, чтобы увидеть детали
2. **Фильтры**: используйте фильтры сверху для поиска конкретной кампании
3. **Сравнение**: сравните эффективность разных источников

**Пример:**

```
utm_source    utm_medium    utm_campaign           Визиты    Конверсия
yandex        cpc           search_ai_base_nov     145       12.4%
telegram      social        channel_promo          89        8.9%
vk            cpc           retargeting_base       67        15.2%
```

---

### 2. Отчёт "Источники трафика"

**Путь:** `Отчёты` → `Стандартные отчёты` → `Источники` → `Источники, сводка`

Здесь показаны все источники (включая прямые заходы, поисковики, utm-трафик):

**Основные группы:**
- **Переходы из поисковых систем** — органический трафик
- **Рекламные системы** — Директ, Google Ads
- **Социальные сети** — VK, Telegram, Instagram
- **Рекомендательные системы** — Дзен, рекомендации
- **Сайты** — рефералы
- **Внутренние переходы** — utm_source из ваших кампаний

---

### 3. Отчёт "Директ, сводка"

**Путь:** `Отчёты` → `Стандартные отчёты` → `Источники` → `Директ, сводка`

**Важно!** Этот отчёт показывает данные **только если Метрика связана с Директом**.

Если связка настроена, вы увидите:
- Кампании из Директа
- Клики и расходы
- Конверсии и CPA (цена конверсии)
- ROI по каждой кампании

## Проверка целей

### 1. Настройка целей

**Путь:** `Настройки` → `Цели` → **Добавить цель**

Создайте следующие цели:

#### Цель 1: Клик по кнопке покупки воркшопа

- **Название**: Клик на покупку воркшопа
- **Тип**: JavaScript-событие
- **Условие**: `workshop_purchase_click`

#### Цель 2: Отправка заявки на вакансию

- **Название**: Заявка на вакансию
- **Тип**: JavaScript-событие
- **Условие**: `job_application_submitted`

#### Цель 3: Просмотр ключевых страниц (опционально)

- **Название**: Просмотр страницы /baza
- **Тип**: Посещение страниц
- **Условие**: `url` содержит `/baza`

---

### 2. Проверка срабатывания целей

**Путь:** `Отчёты` → `Стандартные отчёты` → `Конверсии` → **Выберите цель**

**Что смотреть:**

| Метрика | Описание |
|---------|----------|
| **Достижения цели** | Сколько раз цель сработала |
| **Конверсия** | % визитов с достижением цели |
| **Средняя глубина** | Сколько страниц смотрят перед конверсией |

**Детализация по источникам:**

Переключите группировку на `utm_source` или `utm_campaign`, чтобы увидеть:

```
Кампания                    Визиты    Достижения    Конверсия
search_ai_base_nov          145       18            12.4%
telegram_channel_promo      89        8             9.0%
```

---

### 3. Отчёт "События"

**Путь:** `Отчёты` → `Стандартные отчёты` → `Содержание` → `События`

Здесь видны все JavaScript-события, которые вы отправляете:

**Примеры событий:**

| Событие | Параметры | Описание |
|---------|-----------|----------|
| `workshop_purchase_click` | workshop, tier, price, utm_source... | Клик по кнопке записи на воркшоп |
| `job_application_submitted` | position, utm_source... | Отправка заявки на вакансию |
| `page_view` | page_path, utm_source... | Просмотр страницы |

**Для детального просмотра:**
1. Кликните на событие → увидите параметры
2. Фильтруйте по utm_source, чтобы понять, какой источник приводит к целевым действиям

## Связь Директа с Метрикой

### Зачем связывать?

При связке вы получите:
- **Расходы по кампаниям** прямо в Метрике
- **ROI и CPA** — окупаемость рекламы
- **Детализацию по площадкам** (поиск, РСЯ)
- **Ключевые слова**, которые привели к конверсии

---

### Как связать Директ с Метрикой

#### Шаг 1: Откройте Директ

Перейдите на https://direct.yandex.ru

#### Шаг 2: Добавьте счётчик Метрики

1. Выберите кампанию
2. Настройки → **Счётчики Метрики**
3. Добавьте счётчик **105383064**
4. Сохраните

#### Шаг 3: Включите автометки (рекомендуется)

В настройках кампании:
- **Автометки**: Включено ✅

**Что это даёт:**
- Яндекс автоматически добавляет свои метки (yclid)
- Вы видите детальную статистику по площадкам и объявлениям
- Не нужно вручную прописывать utm_source=yandex

**Важно:** Автометки работают параллельно с вашими UTM. Если вы добавляете свои utm-метки, они тоже сохраняются.

---

### Автометки vs Ручные UTM

| Параметр | Автометки (yclid) | Ручные UTM |
|----------|-------------------|------------|
| **Добавление** | Автоматически | Вручную в каждую ссылку |
| **Данные** | Площадки, объявления, ключевые слова | Источник, кампания, контент |
| **Расходы** | ✅ Показывает | ❌ Не показывает |
| **Гибкость** | ❌ Фиксированная структура | ✅ Любые названия |

**Рекомендация:** Используйте **оба метода**:
- Автометки для статистики Директа
- Ручные UTM для своей аналитики

**Пример ссылки с обоими:**
```
https://ru.sfer.ai/baza?utm_source=yandex&utm_medium=cpc&utm_campaign=base_nov&yclid=123456789
```

## Тестирование перед запуском рекламы

### Чек-лист проверки

#### ✅ 1. Проверьте, что Метрика установлена

**Как проверить:**

1. Откройте сайт https://ru.sfer.ai
2. Нажмите **F12** (DevTools) → **Console**
3. Выполните:
   ```javascript
   ym(105383064, 'getClientID', function(clientID) {
     console.log('Метрика работает! Client ID:', clientID);
   });
   ```

**Ожидаемый результат:**
```
Метрика работает! Client ID: 1234567890123456789
```

Если ошибка — проверьте, что скрипт Метрики загружен в `index.html`.

---

#### ✅ 2. Проверьте сохранение UTM в sessionStorage

1. Откройте сайт с UTM:
   ```
   https://ru.sfer.ai/?utm_source=test&utm_medium=check
   ```

2. В DevTools → **Console**:
   ```javascript
   console.log(sessionStorage.getItem('traffic_params'));
   ```

**Ожидаемый результат:**
```json
{
  "utm_source": "test",
  "utm_medium": "check",
  "referrer": "direct",
  "landing_page": "/",
  "timestamp": "2025-11-18T15:00:00.000Z"
}
```

---

#### ✅ 3. Проверьте отправку целей

1. Откройте страницу воркшопа:
   ```
   https://ru.sfer.ai/baza?utm_source=test
   ```

2. Кликните по кнопке **"Записаться"**

3. В консоли должно появиться:
   ```
   ✅ Метрика: цель достигнута workshop_purchase_click {...}
   ```

4. Проверьте в Метрике → **В реальном времени** → **События**

---

#### ✅ 4. Проверьте передачу UTM в Google Sheets

1. Заполните форму заявки на вакансию с UTM:
   ```
   https://ru.sfer.ai/jobs/pm/apply?utm_source=test
   ```

2. Отправьте форму

3. Откройте Google Sheets — последняя строка должна содержать:
   - `utm_source`: test
   - `utm_medium`: (пусто, если не указан)
   - `referrer`: direct
   - `landing_page`: /jobs/pm/apply

**Важно:** Для этого нужно обновить Google Apps Script (см. `docs/UTM_TRACKING_GUIDE.md`).

## Частые вопросы

### ❓ Метки не появляются в Метрике

**Возможные причины:**

1. **Не прошло достаточно времени**
   - Данные появляются с задержкой 10-30 секунд
   - Проверяйте в режиме реального времени

2. **Счётчик не установлен**
   - Проверьте через DevTools → Network → найдите запрос к `mc.yandex.ru`

3. **AdBlock блокирует Метрику**
   - Отключите AdBlock на время теста

4. **Неправильный формат UTM**
   - Проверьте, что используете `utm_source`, а не `source`
   - Все параметры должны быть в lowercase

---

### ❓ Цели не срабатывают

**Проверьте:**

1. **Цели созданы в Метрике?**
   - Настройки → Цели → должны быть `workshop_purchase_click`, `job_application_submitted`

2. **JavaScript-события отправляются?**
   - Проверьте консоль на наличие логов:
     ```
     ✅ Метрика: цель достигнута ...
     ```

3. **Метрика загружена до отправки события?**
   - Должна загрузиться до первого клика

---

### ❓ В Директе нет данных о конверсиях

**Убедитесь:**

1. **Метрика связана с Директом**
   - В настройках кампании добавлен счётчик 105383064

2. **Цели в Метрике используются Директом**
   - Настройки → Цели → "Использовать в Директе" ✅

3. **Есть конверсии**
   - Проверьте в Метрике, что цели вообще достигаются

4. **Данные импортируются с задержкой**
   - В Директе конверсии появляются через 2-6 часов

## Рекомендации по аналитике

### 1. Настройте сегменты

**Путь:** `Отчёты` → **Создать сегмент**

**Полезные сегменты:**

- **Платные каналы**: utm_medium = cpc OR utm_medium = display
- **Органический трафик**: utm_source не задан AND источник = поисковые системы
- **Telegram-трафик**: utm_source = telegram
- **Конверсии от Директа**: utm_source = yandex AND достигнута цель

---

### 2. Создайте дашборд

Добавьте виджеты:
- **UTM-источники** (топ-5 по визитам)
- **Конверсии по кампаниям** (график)
- **ROI по источникам** (таблица)
- **Воронка**: визит → просмотр /baza → клик на покупку

---

### 3. Настройте email-отчёты

**Путь:** `Настройки` → `Email-отчёты` → **Добавить отчёт**

Получайте еженедельно:
- Источники трафика
- Конверсии по целям
- Сравнение с предыдущей неделей
